# Documentation Structure

This project keeps core architecture files at repository root and organizes the remaining operational docs under `docs/`.

## Keep at Root

- `README.md`
- `ARCHITECTURE.md`
- `DOCUMENTATION.md`
- `plan-crackCmsPlatformFixAndEnhancement.prompt.md`

## Organized Docs

- `docs/setup/` - environment and integration setup guides
- `docs/guides/` - workflow and process guides
- `docs/reports/` - implementation and capacity reports
- `docs/reference/` - reference notes and changelog-style docs
- `docs/backend/` - backend-specific operational docs
